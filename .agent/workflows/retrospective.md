---
description: Retrospective cuối sprint — đánh giá, rút kinh nghiệm, cải tiến quy trình
---

Dùng cuối mỗi sprint để nhìn lại và cải thiện quy trình làm việc.

1. **ĐỌC SPRINT PLAN**: Mở file sprint plan của sprint vừa kết thúc.

2. **ĐÁNH GIÁ KẾT QUẢ**:
   - Bao nhiêu % task đã hoàn thành?
   - Task nào bị trễ hoặc chưa làm?
   - Có task nào phát sinh ngoài kế hoạch không?

3. **PHÂN TÍCH** ba góc độ chính:

   **✅ Điều gì đã làm TỐT?**
   (Những quyết định đúng, workflow hiệu quả, code chất lượng cao)

   **⚠️ Điều gì cần CẢI THIỆN?**
   (Bug tái diễn, quy trình chậm, thiếu tài liệu, test coverage yếu)

   **💡 Điều RÚT RA được?**
   (Bài học kỹ thuật, bài học về quản lý công việc)

4. **ĐỀ XUẤT CẢI TIẾN**:
   - Nếu cần cập nhật workflow → đề xuất thay đổi cụ thể
   - Nếu cần thêm tool/automation → ghi vào backlog

5. **LƯU BÁO CÁO**: Lưu retro vào `docs/sprint/RETRO_SPRINT_[số].md`

6. **ĐẶT MỤC TIÊU**: Đề xuất 1-2 mục tiêu cải tiến cụ thể cho sprint tiếp theo.
